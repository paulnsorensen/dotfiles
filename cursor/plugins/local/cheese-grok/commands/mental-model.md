authored model of the codebase. The artifact is the comprehension
test: if you can't write the section, you don't understand it.

Open `docs/mental-model.md` (create if missing).

If creating: scaffold these sections, leave the bodies empty for me to
fill in:

```markdown

Last updated: YYYY-MM-DD

## One-paragraph elevator pitch

## Building blocks (≤7 boxes I'd draw on a whiteboard)

## Entry points (every way control flow can begin)

## Infrastructure (run locally, ship to prod)

## Open questions / things I'm not sure about
```

If appending: add a dated section at the END titled
`## Session notes — YYYY-MM-DD` and let me fill the body. Do not
overwrite or "tidy up" prior sections.

After saving, list which sections in the document are still empty and
suggest which to tackle next. Do not write the content for me — even
if I ask. This is the comprehension test.
