# ADR session-convergence-003: Wheypoint provenance as additive header fields with explicit join/split verbs  [status: accepted]

- **Context:** Joining (merging two session threads) and splitting (forking one) needs lineage in wheypoint notes, which today carry only `status/next/mode/artifact` + an orientation line. Placement candidates: new header fields vs a body `## Provenance` section. Operation candidates: explicit `/wheypoint --join/--split` verbs vs hand-authored convention.
- **Decision:** Four optional header fields — `session:` (`<harness>:<id>`), `git:` (`<branch>@<short-sha>`, where the `@<short-sha>` suffix is itself optional — branch-only is valid when the sha is unavailable, e.g. a log-reconstructed session), `created:` (UTC ISO-8601), `parents:` (slug list) — between `artifact:` and the orientation line, written/consumed by explicit `--join`/`--split` verbs that auto-fill lineage (join = 2+ parents; split = two children sharing one parent).
- **Alternatives:** Body section (rejected — provenance is part of the machine-read resumption contract `/cheese --continue` acts on, not documentation); convention-only lineage (rejected — no enforcement, silent drift).
- **Consequences:** Safe because handoff parsers are key-based (skills/cheese/SKILL.md §--continue parses named keys; verified 2026-07-09) and all new fields are optional — pre-provenance notes stay valid, and the orientation line remains the first non-key line. Any future *positional* parser of wheypoint headers would break this invariant; don't write one.
